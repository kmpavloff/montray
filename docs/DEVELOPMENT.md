# Development

## Requirements

- Windows 10 or Windows 11.
- .NET 8 SDK or newer.
- Windows PowerShell or Windows Terminal.

Build and hardware validation should run on Windows, not WSL Linux, because the app targets WinForms and Windows hardware sensors.

## Commands

Restore and build:

```powershell
dotnet restore .\Montray.sln
dotnet build .\Montray.sln
```

Run tests:

```powershell
dotnet test .\tests\Montray.Core.Tests\Montray.Core.Tests.csproj
```

Run the same project-wide checks used by CI:

```powershell
dotnet restore .\Montray.sln
dotnet build .\Montray.sln --configuration Release --no-restore
dotnet test .\Montray.sln --configuration Release --no-build
```

Run locally:

```powershell
dotnet run --project .\src\Montray\Montray.csproj
```

For hardware validation, keep the tray app non-elevated and install the sensor service from the tray menu. The menu launches `scripts\install-service.ps1` with UAC and points it at the built `montray-service.exe`.

Manual service commands:

```powershell
.\scripts\install-service.ps1 -ServiceExePath .\src\Montray.Service\bin\Debug\net8.0-windows\montray-service.exe
.\scripts\uninstall-service.ps1
```

Publish a local release build:

```powershell
dotnet publish .\src\Montray\Montray.csproj `
  --configuration Release `
  --runtime win-x64 `
  --self-contained true `
  --output .\artifacts\publish `
  -p:PublishSingleFile=true `
  -p:IncludeNativeLibrariesForSelfExtract=true `
  -p:EnableCompressionInSingleFile=true `
  -p:DebugType=None `
  -p:DebugSymbols=false

dotnet publish .\src\Montray.Service\Montray.Service.csproj `
  --configuration Release `
  --runtime win-x64 `
  --self-contained true `
  --output .\artifacts\publish-service `
  -p:PublishSingleFile=true `
  -p:IncludeNativeLibrariesForSelfExtract=true `
  -p:EnableCompressionInSingleFile=true `
  -p:DebugType=None `
  -p:DebugSymbols=false
```

## Project Structure

- `src/Montray`: WinForms tray app.
- `src/Montray.Core`: normalized sensor models and formatting/selection logic.
- `src/Montray.Hardware`: LibreHardwareMonitor integration.
- `src/Montray.Service`: Windows Service that exposes sensor readings over a local named pipe.
- `tests/Montray.Core.Tests`: unit tests for core behavior.
- `.github/workflows`: CI and release automation.

Keep LibreHardwareMonitor-specific code inside `src/Montray.Hardware`. UI code should consume normalized `SensorReading` values.

## Releases

Releases are created from git tags:

```powershell
git tag v0.1.0
git push origin v0.1.0
```

The release workflow publishes a self-contained `win-x64` zip and attaches it to a GitHub Release. The custom archive contains the tray app, sensor service, service scripts, license files, and installation instructions. GitHub also shows automatic `Source code` archives for every tag; those are generated by GitHub and are not the app distribution.
